# CODING_RULES_ZH（代码风格与中文注释规范）

## 1. 总体要求
- 代码必须面向对象，分层设计，分文件管理
- 禁止把多个核心类写到同一个 .cpp
- 每个模块职责清晰，避免跨层调用

## 2. 命名规范
- 类名：PascalCase（例：AuthService）
- 函数名：camelCase（例：checkCompanyWritePermission）
- 常量：UPPER_SNAKE_CASE（例：MAX_RETRY_TIMES）
- 文件名：模块语义化命名（例：auth_service.h / auth_service.cpp）

## 3. 注释规范（中文优先）
- 每个公开类必须有中文说明
- 每个关键函数必须有中文注释（输入/输出/异常）
- 复杂逻辑分段注释，解释“为什么这么做”

## 4. 输出与日志规范
- 控制台提示尽量中文（如“登录成功”“参数错误”“权限不足”）
- 错误日志至少包含：模块名、函数名、错误原因
- 数据库异常必须打印可定位信息（但不能暴露敏感信息）

## 5. 分层强约束
- Controller：只负责请求参数和响应封装
- Service：只负责业务规则和权限校验
- Repository：只负责数据库读写
- 禁止 Controller 直接写 SQL

## 6. 提交前检查
- 是否补充了中文注释
- 是否遵守分层
- 是否有最小可运行示例
- 是否有可读的错误提示
