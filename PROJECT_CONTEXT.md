# PROJECT_CONTEXT（项目背景与不可变约束）

## 1. 项目背景
- 项目名称：工程公司财务管理系统
- 业务目标：支持公司、项目、成本、营收、税金、利润等核心财务业务
- 使用人群：内部业务人员 + 管理员

## 2. 技术不可变约束（当前阶段）
1. 后端：纯 C/C++
2. 数据库：MySQL
3. 构建方式：Makefile（不使用 CMake）
4. 权限模型：账号密码登录 + 公司级数据权限 + 读写分离权限
5. 架构：Controller / Service / Repository 分层

## 3. 权限业务规则（不可破坏）
1. 没有某公司权限，不能查看该公司任何数据
2. 支持公司级只读和读写权限
3. 同一账号可同时拥有多家公司权限
4. 超级管理员可全量访问（仅限受控管理员账号）

## 4. 开发原则
- 优先保证可读性和可维护性
- 优先中文注释和中文错误提示
- 复杂功能先做最小可运行版本，再迭代

## 5. 变更约定
- 凡涉及上述“不可变约束”的变更，必须先更新本文件并记录理由
